# データベース設計（SQLite）

最終更新: 2026-07-01

正本（source of truth）は Native Messagingホスト側のSQLiteのみ（`docs/仕様書.md` 4.1節）。DDLは [`crates/engine-core/fixtures/schema.sql`](../crates/engine-core/fixtures/schema.sql)、サンプルデータは [`crates/engine-core/fixtures/seed.sql`](../crates/engine-core/fixtures/seed.sql) を参照。本ドキュメントは各テーブルの役割と関係を説明する。

接続時は必ず `PRAGMA foreign_keys = ON;` を実行すること。

---

## テーブル一覧

| テーブル                                     | 役割                                 |
|------------------------------------------|------------------------------------|
| `app_settings`                           | キー・バリューの単一設定（保存先ベースパス、アプリバージョン等）   |
| `courses`                                | Moodleのコース一覧                       |
| `global_rule`                            | グローバルなフォルダ命名ルール（常に1行）              |
| `course_rule_overrides`                  | コース単位の例外ルール（回ごとに分けない等）             |
| `files`                                  | 保存済みファイルのメタデータ・ハッシュ・ルール適合状況        |
| `duplicate_groups` / `duplicate_members` | 重複・類似ファイルのグループ化                    |
| `assignments`                            | 課題・締切情報（ファイル/Moodleテキスト/ダッシュボード由来） |
| `notification_rules`                     | 締切通知のタイミング設定（複数登録可）                |
| `search_index_meta`                      | Tantivy全文索引の補助メタ情報                 |

## 関係の要点

- `files.course_id` は `courses.id` を参照。手動保存などコース不明のファイルは `NULL` を許容
- `course_rule_overrides` は `courses` と1対1（`UNIQUE(course_id)`）。行が無いコースは `global_rule` のみ適用
- `files.rule_compliant = 0` のレコードが「ルール管理・整合性チェック画面」の警告一覧の元データ。理由は `violation_reason` に保持する。違反検出ロジックは `crates/engine-core` の `RuleEngine` が担当し、専用テーブルは持たずファイル行への注釈で表現する
- `assignments.due_at_status = 'needs_review'` が期限の異常検知（学期から大きく外れた締切）の対象
- `assignments.submission_mode` は提出状況の扱い方を表す：`moodle_auto`（Moodleから自動反映）／`manual`（ユーザーが手動で印）／`notify_only`（提出状況は追跡せず通知のみ）
- `duplicate_groups` は完全重複（`method='exact'`、`hash_blake3`一致）と類似（`method='similar'`、simhash近傍）の両方を表現できる
- `notification_rules` は複数行登録でき、ユーザーが「3日前」「当日」等を個別に有効/無効化できる

## サンプルデータの内容

`seed.sql` は Fuzzy.pdf のダッシュボードモックアップに登場する6科目（情報アーキテクチャ・データベース・離散数学・アプリ演習・認知科学概論・英語IIB）を使用し、以下のケースを含む。

- コース単位の例外ルール例：「アプリ演習」を回ごとに分けない設定
- ルール違反ファイルの例：規定外の場所に保存された資料
- 完全重複ファイルの例：同一資料を誤って2回保存したケース
- 締切の異常検知例：学期の範囲から外れた締切（`needs_review`）
- 期限切れの例：通知のみ対象の締切が過ぎたケース

`python3` の標準 `sqlite3` モジュールで読み込み検証済み（外部キー制約も含めてエラーなし）。
