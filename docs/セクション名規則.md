# セクション名認識規則

最終更新: 2026-07-18

この文書は、既存フォルダ構成とファイル名から授業の回・週等を認識する規則を定める。実装上の正本は`crates/engine-core/src/section.rs`とし、規則を変更する場合はこの文書とテストを同時に更新する。

## 基本方針

- フォルダ名は文字列全体が規則に一致した場合だけセクションとして扱う
- ファイル名は先頭にある明示的なセクション表記、または最初の`_`、`-`、空白までの数字を接頭辞として判定する
- 半角・全角アラビア数字と一般的な漢数字を数値へ正規化する
- 単に名前の途中に数字が含まれるだけの場合はセクションとして扱わない
- 認識結果は保存パターンの候補提示にだけ使い、ファイルを自動で移動・削除しない

## 認識する表記

| 規則ID | 表記 | 例 | 正規化例 |
|---|---|---|---|
| `ja_ordinal` | `第` + 数字 + `回/週/講/章` | `第十二回` | `第12回` |
| `ja_suffixed` | 数字 + `回/週/講/章` | `二十週` | `第20週` |
| `en_week` | `week` + 数字 | `Week 4` | `week 4` |
| `en_unit` | `unit` + 数字 | `Unit２` | `unit 2` |
| `en_lesson` | `lesson` + 数字 | `Lesson-3` | `lesson 3` |
| `en_lecture` | `lecture` + 数字 | `Lecture_5` | `lecture 5` |
| `en_session` | `session` + 数字 | `Session6` | `session 6` |
| `numeric_file_prefix` | ファイル名先頭の数字 | `09_資料.pdf` | `9` |

英語キーワードと数字の間には、空白・`_`・`-`を使用できる。英字の大文字と小文字は区別しない。

## 数字の表記

### アラビア数字

- 半角: `0`〜`9`
- 全角: `０`〜`９`
- 半角と全角の混在も数値として扱う

### 漢数字

- 数字: `〇`、`零`、`一`、`二`、`三`、`四`、`五`、`六`、`七`、`八`、`九`
- 単位: `十`、`百`、`千`
- 単位なしの連続表記: `二〇` → `20`
- 単位付き表記: `十二` → `12`、`百一` → `101`

`万`以上の単位、旧字体、`二三十`や`十百`のように解釈が曖昧・不正な表記は認識しない。

## 認識しない例

| 表記 | 理由 |
|---|---|
| `2026前期` | 回・週等を示す単位がないフォルダ名 |
| `資料4` | 数字が名前の末尾にあるだけ |
| `データベース2` | 科目名の一部である可能性が高い |
| `第一万回` | 対応範囲外の数値単位を含む |
| `講義資料` | セクション番号がない |

## 数字だけのファイル名接頭辞に関する注意

`09_資料.pdf`のような接頭辞は既存ファイルの命名規則を推定する手掛かりになる一方、数字だけでは意味を確定できない。たとえば次の値をセクション番号と誤認する可能性がある。

- `2026_講義概要.pdf`の年度
- `20260713_配布資料.pdf`の日付
- `01_表紙.pdf`の並び順や資料番号
- `2_改訂版.pdf`のバージョン番号
- 学籍番号、科目コード、ページ番号

このため、数字だけの接頭辞はセクションの確定値として自動適用せず、複数ファイルから得た支持率、親フォルダ名、Moodle上のセクション情報と合わせて候補提示にだけ使用する。

## 規則追加時の確認

1. `crates/engine-core/src/section.rs`に認識処理と安定した規則IDを追加する
2. `crates/engine-core/tests/section_names.rs`に認識例と非認識例を追加する
3. 本文書の対応表と境界条件を更新する
4. 数字を含む年度・日付・科目名が誤認識されないことを確認する
