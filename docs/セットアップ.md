# 開発環境セットアップ（Windows 11）

Fuzzy は Windows 11 専用アプリです。このドキュメントでは、開発に必要なツール（Bun / Rust / Tauri 前提パッケージ）のインストール方法と、リポジトリの初回セットアップ手順をまとめます。

---

## 1. Bun のインストール

PowerShell を開いて以下を実行する。

```powershell
powershell -c "irm bun.sh/install.ps1|iex"
```

確認：

```powershell
bun --version
```

`bun: コマンドが見つかりません` のようなエラーが出る場合は、ターミナルを開き直してください。

他の方法（任意）：`npm install -g bun` / `scoop install bun`

参考：[Bun公式インストールガイド](https://bun.com/docs/installation)

---

## 2. Rust のインストール（rustup）

以下のいずれかの方法でインストールする。

**方法A：winget**

```powershell
winget install -e --id Rustlang.Rustup
```

**方法B：公式インストーラー**

[rustup.rs](https://rustup.rs/) から `rustup-init.exe` をダウンロードして実行する。

インストール中、`default host triple` が `x86_64-pc-windows-msvc`（MSVCツールチェーン）になっていることを確認する。すでにRustが入っている場合は、念のため以下を実行しておく。

```powershell
rustup default stable-msvc
```

確認：

```powershell
rustc --version
cargo --version
```

**インストール・切り替え後はターミナル（場合によってはPC）を再起動すること。**

参考：[rustup公式](https://rust-lang.github.io/rustup/installation/windows-msvc.html)

---

## 3. Tauri の前提パッケージ（Windows）

Tauri は Windows 上での開発に **Microsoft C++ Build Tools** と **WebView2** を必要とする。

### 3.1 Microsoft C++ Build Tools

1. [Microsoft C++ Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) をダウンロードして実行
2. インストール時に **「C++ によるデスクトップ開発」** ワークロードにチェックを入れる

### 3.2 WebView2 ランタイム

Windows 10（バージョン1803以降）・Windows 11には標準搭載済みのため、通常は追加作業不要。万が一入っていない場合は [WebView2 Runtime ダウンロードページ](https://developer.microsoft.com/en-us/microsoft-edge/webview2/#download-section)から「Evergreen Bootstrapper」を入手してインストールする。

### 3.3 VBSCRIPT（MSIインストーラーを作る場合のみ）

`tauri.conf.json` で `msi` ターゲットのビルドを行う場合のみ必要。`failed to run light.exe` のようなエラーが出た場合に確認する。

設定 → アプリ → オプション機能 → Windows のその他の機能 → **VBSCRIPT** が有効になっているか確認する。

参考：[Tauri公式 Prerequisites（Windows）](https://v2.tauri.app/start/prerequisites/)

## 4. VS Code

`.vscode/extensions.json` に推奨拡張機能が定義されているため、VS Codeでリポジトリを開くと自動でインストールが提案される。インデントは **タブ**、改行コードは **LF** に統一されているので、エディタ側の自動整形設定を上書きしないよう注意する（`.editorconfig` / `.vscode/settings.json` が自動適用される）。

---

## 5. リポジトリのセットアップ

1. リポジトリを clone する
2. ルートで依存をインストール：`bun install`
	- パッケージ管理は **bun に統一**（ロックファイルは `bun.lock`。npm/yarn での install は行わない）
	- 初回は postinstall（`wxt prepare` による型生成）が実行される（ルート `package.json` の `trustedDependencies` で許可済み）
3. Rust側のビルド確認：`cargo build`（ルートのCargoワークスペースが `crates/engine-core`・`apps/native-host`・`apps/desktop/src-tauri` をまとめてビルドする）
4. 整形・型チェック：`bun run check` / `bun run build`

※各アプリ・クレートはPhase0（#32〜#35）で生成済みのため、`bun create` / `cargo new` による生成作業は不要。`apps/desktop/src-tauri` と `apps/native-host` を `crates/engine-core` に依存させる作業はPhase1（#36〜）で行う。

---

## 6. 動作確認

- **Native Messagingホスト**：`cargo build -p native-host`でビルドする。拡張機能からの実応答確認には、ホストマニフェスト（JSON）の実行パスをビルドした`native-host.exe`へ向け、Fuzzy拡張機能のIDだけを許可元としてWindowsへ登録しておく必要がある。未登録の場合、初期セットアップは完了しない
- **初期セットアップアプリ（Tauri）**：`apps/desktop` 内で `bun run tauri dev`を実行する。起動前に`apps/extension`が自動ビルドされ、同梱用成果物が`apps/extension/.output/chrome-mv3`へ生成される
- **拡張機能**：Tauriの最終ステップで「同梱フォルダーを表示」を押し、Native Messagingを利用できるManifest V3対応ブラウザの拡張機能管理画面から表示されたフォルダーを読み込む。ブラウザ名の選択は不要
- **完了確認**：拡張機能のインストール・更新・ブラウザ起動時に`reportExtensionRuntime`が送信され、native-hostがSQLiteへ受信日時とバージョンを保存する。Tauriは今回の起動後に届いた互換性のある応答を確認すると自動的に完了表示へ切り替わる。チェックボックスやlocalStorageの完了値は使用しない

ホスト登録先はブラウザ実装ごとに異なるが、初期セットアップ画面はブラウザを判定しない。登録済みのNative Messaging経路から正しいFuzzy拡張機能の応答が届くかだけを確認する。本番インストーラーでのホスト登録方法は配布設定と合わせて確定する。

---

## 7. トラブルシューティング

| 症状                                | 対処                                      |
|-----------------------------------|-----------------------------------------|
| `bun` コマンドが見つからない                 | ターミナル再起動／PATH確認（1章参照）                   |
| `cargo build` が C++ 関連のエラーで失敗する   | Build Tools の「C++ によるデスクトップ開発」が入っているか確認 |
| Tauriアプリ起動時にWebView関連のエラー         | WebView2 Runtimeが入っているか確認（3.2章）         |
| 拡張機能を読み込んでも初期セットアップが完了しない | native-hostをビルド済みか、ホストマニフェストの実行パスと許可する拡張機能IDが正しいか確認する |
| MSIビルドで `failed to run light.exe` | VBSCRIPT機能が有効か確認（3.3章）                  |

困った場合は [Tauri公式トラブルシューティング](https://v2.tauri.app/develop/debug/) も参照する。
