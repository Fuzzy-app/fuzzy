# Issue #38 作業レポート

作成日: 2026-07-13

## タスク

`crates/engine-core` の `ScanEngine` に、選択フォルダの再帰走査と既存ファイルからの保存パターン推定を実装する。

完了条件は次の3点。

- 指定フォルダを再帰走査してファイル一覧を取得する
- 命名・階層に規則性がある場合にパターン候補を確からしさ順で返す
- 既存の6科目サンプル構成に対する推定結果をユニットテストで確認する

## 解決方法

### 再帰走査

- 走査起点の存在とフォルダ種別を検証し、不正な場合は `EngineError::InvalidPath` を返す
- 走査起点を絶対パスへ正規化し、通常ファイルのパス・名前・サイズ・更新日時を収集する
- ジャンクションやシンボリックリンクによる循環を避けるため、リンクは追跡しない
- 呼び出しごとに結果順が変わらないよう、絶対パス順に並べる

### 保存パターン推定

- `第N回`、`第N週`、`N回`、`N週`、`week N`、`unit N`、`lesson N` をセクションフォルダとして認識する
- 各ファイルが支持するレイアウトを集計し、全ファイルに対する支持率を `confidence` として返す
- 3ファイル以上を分析する場合は2件以上の支持を候補条件とし、偶然の1件を規則として提示しない
- 候補は `confidence`、一致件数、テンプレート名の順で決定的に並べる
- 1ファイルだけでは規則性を判断できないため、候補を返さない

実装は読み取りと候補提示だけを行う。既存ファイルの移動・削除・変更は行わない。

## 検証

- 再帰走査でファイル情報と決定的な順序が得られること
- ファイルを走査起点に指定した場合にエラーになること
- 6科目サンプルのうち、回別フォルダを使う5科目から `{course}/{section}/{filename}` を支持率 `5/6` で推定すること
- 1ファイルだけの構成からパターンを過剰推定しないこと

## 不足情報・未決事項

`docs/仕様書.md` 6章にあるとおり、保存パターン推定の正式な判定基準は未決定である。本実装では上記の保守的な基準を暫定採用した。今後、次の点が決まった場合は推定器の拡張が必要になる。

- パターン候補として提示する最低支持率
- `2026前期` のように年度と学期を結合したフォルダ名のトークン表現
- `第1章`、任意のMoodleセクション名など、回・週以外をセクションとみなす範囲
- ファイル名の規則性を、フォルダ構成とは別のルールとして保持するか

これらは今回の完了条件を妨げないが、UIで候補を確定して永続化する前に仕様化することが望ましい。
