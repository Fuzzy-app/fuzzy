# Fuzzy 詳細仕様（ドラフト v0.4）

最終更新: 2026-07-15

Moodleの授業資料を自動整理し、課題・締切を一元化する学習補助アプリ。新規ダウンロード時の保存先推薦と、保存済み資料・課題情報の活用を担う。**保存済みファイルを自動で動かす・削除することは行わない**（推薦・提示・警告のみ、実行はすべてユーザー操作）。

DB詳細スキーマ・API契約は別セッションで設計する。

---

## 1. 機能要件

### 1.1 資料を保存する

- **Moodle情報の取得**：コース名・回（セクション）・Moodle上の階層を読み取る（コース直下に限らず深い階層の資料も対象）。あわせてMoodleページのテキスト本文やダッシュボードのウィジェットも取得し、ファイルに紐付かない課題・締切情報も拾う。認証が必要な資料の取得はログイン済みの拡張機能が担い、native-hostはブラウザ認証情報を保持しない
- **保存先の提案と選択**：適切と思われる保存先候補を確からしさ順に複数提示。ユーザーは候補から選ぶか、自分で場所を指定して保存できる。保存操作後は、確認済みの保存先に必要なフォルダを自動作成してよい
- **一括保存機能**：複数の資料をまとめて選択し、一度に保存
- **ZIPファイルの解凍機能**：展開して保存するかを提案。中身が二重フォルダ等の無駄な入れ子になっている場合は簡略化を提案
- **類似ファイルの検索機能**：すでに似た資料が保存されている場合、保存前にその可能性を通知し重複を防ぐ

### 1.2 整理方法の決定

**初期セットアップ（Tauriネイティブ画面・初回のみ）**

- 保存先フォルダの選択（OSネイティブダイアログで実パスを取得）
- 選択フォルダ内の既存ファイル構成を再帰スキャンし、近い「フォルダの作り方」候補を提示。整理ルールを持たないユーザーにはおすすめの作り方を提示
- **既存ファイルからの保存パターン推定**：選択フォルダに既に保存されている資料の命名・配置に予測可能な規則性がある場合、それをそのままルールとして保持する
- グローバルなフォルダ命名ルールの選択・作成

**ルール管理・整合性チェック画面（拡張機能側・常設）**

- グローバルルールの編集
- **コース単位の例外ルール**：「このコースは回ごとに保存しない」のような、特定コースだけのオーバーライド設定
- ルール更新はSQLite上のコースIDを基準に行い、コース名はクライアント入力として重複保存しない。保存パターンは既知トークンだけを使った相対パスに限定し、絶対パスや `..` を許可しない
- **ルール違反・未整理ファイルの警告**：保持しているルール（初期セットアップ時の推定ルールやカスタムルール）から外れた保存・命名をしているファイルを一覧表示し、ユーザーに修正を促す

### 1.3 保存した資料・課題を活用する

- **ワード検索機能**：入力した語が「どの資料の、どのあたり（ページなど）に書かれているか」を見つけて提示する。答えを生成するのではなく情報の所在を特定する。検索結果から該当箇所（該当ページ等）へ直接ジャンプできる
- **課題・締切のまとめ**：Moodleや資料の中に散らばった課題の締切を集約し、授業名つきで一覧表示
- **提出状況の扱い**：Moodleから提出状況が分かるものは自動反映。Moodle外で提出するものはユーザーが手動で印を付けるか、提出状況は追跡せず期限の通知のみ行う
- **期限の異常検知**：学期の期間から大きく外れた締切（前年度の残りや設定ミスなど）を「要確認」として区別し、もっともらしい期限の候補を提示
  - **データ取得通知**：Moodleから課題・締切データを取得（同期）するたびにブラウザ通知を出す。異常検知の判定は取得のたびに再実行されるため、「取得完了＝最新の判定結果が揃った」ことをユーザーに知らせる
  - **変更点の表示**：取得のたびに前回取得時点のデータと比較し、差分（新規追加・締切変更・要確認への変更・削除された課題など）を検出する。差分が1件以上あればデータ取得通知に変更件数を含め、締切ハブ側で「どの課題の何が、どう変わったか」（変更前後の値）を一覧表示できるようにする
- **期限切れのまとめ**：すでに期限を過ぎた課題もまとめて確認できる
- **学習状況の一覧（ダッシュボード）**：コースごとに、集まっている資料の件数・整理の状況・次の締切をまとめて確認できる。Moodleがダウンしている間もローカルキャッシュから表示できる
- **カレンダー連携**：期限が確定している締切を、提出済み・期限切れも履歴として含めてICSファイルへ書き出す。`due_at_status = 'needs_review'` は誤登録を避けるため除外する。Googleカレンダーへの直接追加は、ユーザーが対象を選び、外部送信を明示的に許可した場合に限る
- **Google ToDo連携（将来機能）**：期限が確定している課題をGoogle ToDo（Google Tasks）へ追加できるようにする。連携は初期状態で無効とし、ユーザーによるGoogle認証、送信対象の確認、明示的な追加操作を必須とする。認証解除と連携停止をいつでも行えるようにし、許可なく課題を外部送信しない
- **締切通知**：締切が近づいたらブラウザ通知で知らせる。3日前・1日前などのプリセットに加え、ユーザーが締切からの任意の相対時間を分・時間・日単位で複数追加・削除し、個別に有効化できるようにする。設定範囲は締切時刻から365日前までとし、表示名は保存した分数から自動生成する
- **デスクトップ通知連携（将来機能）**：現在のブラウザ通知に加え、将来はWindows通知センターへデスクトップ通知を届けられるようにする。ブラウザを完全終了した場合の常駐方式・起動条件・再通知方針は実装Issueで決定し、ユーザーが通知を無効化できるようにする
- **スマートフォン通知**：Fuzzy独自のスマートフォン向けプッシュ通知は実装しない。スマートフォンでの確認・通知は、ユーザーが明示的に許可したGoogleカレンダー／Google Tasks連携先の機能を利用する
- **重複ファイルの検出・提示**：完全重複・類似ファイルを検出して一覧提示する。自動削除は行わず、削除や統合はユーザー操作に委ねる
- **データのエクスポート／インポート**：他PCへの移行や万一の障害に備え、ルール・メタデータをエクスポート／インポートできる（詳細は4章）

## 3. システムアーキテクチャ

### 3.1 全体構成

```
[Moodleページ]
     ⇅ DOM解析（ファイルリンク／本文テキスト／ダッシュボード）
[ブラウザ拡張機能] ──Native Messaging(connectNative, Moodleアクセス中のみ維持)──> [Native Messagingホスト(Rust)]
     │                                                                              │
     └─ IndexedDB（表示用キャッシュ）                                                ├─ SQLite（ルール・メタ・ハッシュ・設定）
                                                                                     └─ Tantivy（全文索引）

[初期セットアップ時のみ] Tauriネイティブアプリ ──OSフォルダダイアログ(実パス取得)──> 共有エンジンを通じて登録

[ユーザーが明示許可した場合のみ] ブラウザ拡張機能 ──Google認証・選択した課題の送信──> Googleカレンダー / Google Tasks
[将来] Native Messagingホスト等 ──ユーザー設定に従った通知──> Windows通知センター
```

### 3.2 コンポーネント別役割

| コンポーネント             | 役割                                                                        | 画面                 | 使用技術                                                 |
|---------------------|---------------------------------------------------------------------------|--------------------|------------------------------------------------------|
| ブラウザ拡張機能            | Moodle監視・DOM解析、保存先提案、一括DL、検索UI、締切ハブ、ダッシュボード、ルール管理、通知、カレンダー連携、オフラインキャッシュ表示 | あり（初期セットアップ以外の全画面） | TypeScript, WXT(Manifest V3), Svelte, Bun, IndexedDB |
| Tauriネイティブアプリ       | 初期セットアップ専用：フォルダ選択、既存構造スキャン、保存パターン推定、初期ルール選択                               | あり（セットアップのみ）       | Tauri, Svelte                                        |
| Native Messagingホスト | 保存・フォルダ生成・再帰走査・全文検索・重複検出・ルール照合・集計・SQLite管理                                | なし（バックグラウンド処理）     | Rust, Tantivy, SQLite, blake3, simhash/LSH, pdfium   |
| 拡張⇄エンジン連携           | 情報・操作要求の受け渡し                                                              | -                  | Native Messaging, ts-rs生成の共有型                        |

### 3.3 共有バックエンド設計

初期セットアップ用のTauriバイナリと、常駐するNative Messagingホストバイナリは別プロセスとして起動するが、フォルダ再帰走査・保存パターン推定・ルール照合・ハッシュ計算・重複検出といった中核ロジックは両者で共通である。これらは独立したRustライブラリクレート（共有エンジンクレート）に切り出し、`ScanEngine`（再帰走査・パターン推定）、`RuleEngine`（グローバル／コース別ルールの照合・違反検出）、`IndexEngine`（Tantivy索引の構築・検索）、`DuplicateDetector`（ハッシュ・類似度による重複検出）のようにトレイト境界で責務を分離する。Tauriバイナリ・Native
Messagingホストバイナリの双方がこの共有クレートに依存する構成とし、ロジックの二重実装を避ける。

### 3.4 Native Messaging 運用方針

- 起動トリガー：Moodleドメインのタブが存在・アクティブな間
- 接続方式：基本は`connectNative`でプロセスを維持し、検索やダッシュボード表示などの頻繁なやり取りに対応。設定変更等の単発操作は`sendNativeMessage`でも可
- 終了：Moodleタブが全て閉じられる、または一定時間操作がない場合に接続を切りホストプロセスを終了

## 4. データ設計と移行方針

### 4.1 役割分担

- **SQLite（Native Messagingホスト側）が正本（source of truth）**：グローバル／コース別ルール、ファイルメタデータ・ハッシュ、全文索引への参照、通知設定など、永続化すべきデータはすべてSQLiteに保存する
- **IndexedDB（拡張機能側）は表示用キャッシュ専用**：Moodleから取得した直近のコース一覧・締切・ダッシュボード集計などを一時的に保持し、Moodleダウン時のオフライン表示に使う。常に再生成可能な複製であり、正本ではない
- IndexedDBが扱うデータ（Moodle由来の表示用スナップショット）とSQLiteが扱うデータ（ファイルシステム由来の構造化データ・ルール）は性質が異なり、同じデータを双方で書き込むことは想定しないため、書き込み競合は基本的に発生しない設計とする

### 4.2 他PCへの移行

- **アプリ本体**：Tauriの標準バンドラで生成する通常のWindowsインストーラー（.msi/.exe）で配布・インストールする。追加の専用インストーラーは不要
- **データ移行**：設定画面の「エクスポート／インポート」から、SQLiteデータベース（ルール・メタデータ・設定一式）をファイルとして書き出し／読み込みできるようにする
- **全文索引（Tantivy）**：索引ファイルはバイナリ依存度が高く単純コピーでの互換性を保証しづらいため、移行時には同梱せず、インポート後に対象フォルダを再スキャンして再構築する
- **IndexedDBキャッシュ**：移行不要。新しいPCでMoodleに再アクセスした時点で自然に再生成される

## 5. 非機能要件

- 対応ブラウザ：拡張機能（Manifest V3）が利用できるブラウザ。和歌山大学Moodleの年度ごとのDOM変更に耐えられるよう、特定年度の固定構造だけに依存しない
- 対応OS：Windows 11のみ
- データは原則ローカル完結、外部送信なし。Googleカレンダー／Google Tasks連携だけは、ユーザーが認証し、対象を確認して明示的に許可した操作に限って例外とする
- Fuzzy独自のスマートフォン向けプッシュ通知基盤は持たない
- 自動的なファイル移動・削除は一切行わない（推薦・提示・警告のみ）
